Current status

The first I should mention is that this project (also known as lablqt) is not bindings as people usually understand it. It was going to be bindings but I realised to generate code for widgets is not very easy and the result will not be convenient enough to deal with (is short words: too many inheritance in Qt Widgets). Also, Qt5 has been realised where Qt Widgets have been claimed as obsolete and a modern for declaratively creating GUI recommended. It uses QML markup to declare GUI, ECMAscript for simple scripting GUI-related actions for controls and C++ for implemeting speed-sensitive business-logic.
At the current moment I think that with my project it is to replace C++ by OCaml (for small application). For big ones some improvements probably not because complex controls could be needed which are inconvinient to implement in pure QML+ECMAScript. I need to pick some big GUI project and test it.

Lablqt in OPAM

N.B.You need Qt5(>=5.0.0) to compile. Please be sure that dev packages are installed too. 
opam remote add lablqt http://kakadu.github.io/lablqt/opam
opam update
opam install mocml         # code generator tool and library
opam install qocamlbrowser # Demo application
