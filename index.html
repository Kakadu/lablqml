Current status

The first I should mention is that this project (also known as lablqt) is not bindings as people usually understand it. It was going to be bindings but I realised it is not very easy to generate code for all widgets and the result will not be convenient enough to deal with (in short: too many inheritance in Qt widgets). Also, Qt5 has been released where Qt Widgets seems to be claimed obsolete and a modern approach for creating GUI declaratively is recommended. It uses QML markup to declare GUI, ECMAscript for simple scripting GUI-related actions for GUI controls, and C++ for implemeting speed-sensitive business logic.
At the current moment I think that with my project is well enough for replacing C++ by OCaml in small applications. In large GUI applications we will probably need to implement some GUI controls in compilable language (like C++). My library doesn't allow implementing these controls in OCaml at this moment. But when I will know in what cases this complex controls are need I'm going to improve library. Maybe I need to pick some big GUI project and rewrite GUI for it.

Lablqt in OPAM

N.B.You need Qt5(>=5.2.0) to compile. Please, be sure that dev packages are installed too. 
opam remote add lablqt http://kakadu.github.io/lablqt/opam
opam update
opam install mocml         # code generator tool and library
opam install qocamlbrowser # Demo application
Development versions: 
opam remote add lablqt http://kakadu.github.io/lablqt/opam-dev
